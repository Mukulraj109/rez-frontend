CODE PATTERNS USED
================================================================================

Pattern 1: Simple Toast with Product Name
─────────────────────────────────────────
try {
  await action(product);
  showSuccess(`${product.name} action completed`);
} catch (error) {
  showError(`Failed to action ${product.name}`);
}

Pattern 2: Conditional Success Messages
───────────────────────────────────────
try {
  if (condition) {
    showSuccess('Message 1');
  } else {
    showSuccess('Message 2');
  }
} catch (error) {
  showError('Error message');
}

Pattern 3: Stock Validation
──────────────────────────
if (quantityInCart < stock) {
  await updateQuantity();
  showSuccess('Updated successfully');
} else {
  showError(`Maximum quantity reached`);
}

================================================================================
SUPPORT & TROUBLESHOOTING
================================================================================

Common Issues:

Issue: Toast not showing
Solution:
- Check ToastProvider wraps entire app in _layout.tsx
- Verify component is inside ToastProvider
- Check import path is correct
- Review console for errors

Issue: Multiple toasts appearing at once
Solution:
- This is normal - queue system handles them sequentially
- Use dismissAll() if needed to clear queue

Issue: Product name showing as undefined
Solution:
- Verify product object has 'name' property
- Check product data is loaded before rendering
- Use optional chaining: product?.name

More help in TOAST_QUICK_REFERENCE.md

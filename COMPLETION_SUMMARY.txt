HOW IT WORKS
================================================================================

LOADING STATE FLOW:
1. User navigates to Store Visit page
2. Component mounts with loading = true
3. StoreVisitLoadingSkeleton renders immediately
4. Shimmer animation starts automatically (continuous)
5. API call: fetchStoreDetails()
6. Data returns after approximately 1-2 seconds
7. loading = false
8. Skeleton unmounts, real content displays

SKELETON DIMENSIONS:
- Header: Full width x 120px (includes padding)
- Cards: Full width x approximately 180px average
- Input fields: Full width x 44px each
- Date cards: 75px x 80px (horizontal scroll)
- Time slots: 30% width x 44px (grid layout)
- Buttons: Full width x 48-54px

================================================================================
TROUBLESHOOTING GUIDE
================================================================================

ISSUE: Skeleton not appearing
SOLUTION: Verify loading = true, check import path, restart dev server

ISSUE: Animation not smooth/jerky
SOLUTION: Verify useNativeDriver: true in SkeletonLoader, test on device

ISSUE: Import errors for skeleton
SOLUTION: Ensure path alias @/components works, check file exists

ISSUE: Layout does not match actual content
SOLUTION: Update SkeletonLoader dimensions to match real component sizes

ISSUE: Skeleton shows too long/too short
SOLUTION: Check API response time, adjust timeout values

ISSUE: Colors do not match
SOLUTION: Update gradient colors in StoreVisitLoadingSkeleton.tsx header

ISSUE: Buttons not visible at bottom
SOLUTION: Check bottomActions positioning, verify paddingBottom value
